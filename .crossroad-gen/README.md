# Crossroad Trainer

ML-based crossroad turn detector for mycc. Trains a DistilBERT encoder classifier to detect when an LLM changes direction mid-response, then exports an ONNX model for streaming inference.

## Overview

mycc's "crossroad" feature detects when an LLM reverses course mid-response (turning words like "However", "Wait", "但"). This subproject replaces regex-based detection with an ML encoder classifier using `distilbert-base-multilingual-cased`.

```
                                                                        │

seed-negative.jsonl ─────────────────────────────────────────────────────┘
                                                                        │
                                                                        ▼

                                                                        │
                                                                        ▼
                                          ~/.mycc-store/crossroad-model/
```


## Setup

### Install dependencies

```bash
cd ~/.mycc-store/crossroad-trainer
pip install -r requirements.txt
```

### Install Ollama (for auto-labeling)

1. Install Ollama from https://ollama.com
2. Pull a model: `ollama pull glm-5:cloud` (or your preferred model)
3. Ensure Ollama is running: `ollama serve`

## Usage

### 1. Collect data (optional)

Add entries to `data/auto-collected.jsonl`. Each line:
```json
{"text": "Let me fix this bug by... Wait, actually the root cause is...", "turnIndex": 35, "source": "session-abc"}
```

### 2. Auto-label collected data

```bash
python label.py                          # Uses default model glm-5:cloud
python label.py --model glm-5:cloud      # Specify model
python label.py --batch-size 20          # Larger batches
python label.py --delay 2.0             # Slower rate to avoid Ollama overload
```

label.py sends each entry to the local Ollama LLM, asking whether the text has a genuine semantic turn at `turnIndex`. Results are written to `data/positive.jsonl` or `data/negative.jsonl`.

### 3. Train the model

```bash
python train.py                           # Default: 4 epochs, batch=16, lr=2e-5
python train.py --epochs 5               # More epochs
python train.py --batch-size 32          # Larger batch
python train.py --skip-eval              # Skip evaluation after training
```

Training reads `seed-positive.jsonl` + `seed-negative.jsonl` + `positive.jsonl` + `negative.jsonl`, fine-tunes `distilbert-base-multilingual-cased`, and exports an ONNX model to `~/.mycc-store/crossroad-model/`. By default the model is **unquantized** (~541 MB) because `quantize_dynamic` crashes with `ShapeInferenceError` on DistilBERT (see `train.py` docstring). Pass `--quantize` to attempt quantization anyway.

### 4. Evaluate

```bash
python evaluate.py                        # Uses default model dir and test cases
python evaluate.py --threshold 0.75      # Adjust turn probability threshold
python evaluate.py --model-dir ~/.mycc-store/crossroad-model
```

Outputs per-file metrics, overall accuracy/F1/precision/recall, confusion matrix, and error analysis report.

### 5. Run tests

```bash
cd ~/.mycc-store/crossroad-trainer
python -m pytest tests/ -v
```

## ONNX Model Interface Contract

The exported ONNX model matches this interface (the agreement with the mycc TS side):

- **Inputs**: `input_ids` (int64, shape `[1, seq_len]`), `attention_mask` (int64, shape `[1, seq_len]`)
- **Output**: `logits` (float32, shape `[1, 2]`) — index 0 = P(no turn), index 1 = P(turn)

The mycc TS side will:
1. Tokenize accumulated streaming text using `tokenizer.json`
2. Feed `input_ids` + `attention_mask` to the ONNX session
3. Apply softmax to logits, take index 1 as P(turn)
4. If P(turn) > threshold (from `config.json`), trigger crossroad

## Version Contract (Python ↔ TypeScript)

The ONNX model and tokenizer are produced by the Python deps in `requirements.txt` and consumed by the mycc TypeScript runtime (`src/loop/crossroad-encoder.ts`). The two sides must stay compatible:

| Component | Python (this repo) | TypeScript (mycc runtime) |
|-----------|-------------------|--------------------------|
| ONNX runtime | `onnxruntime==1.27.0` | `onnxruntime-node ^1.22.0` |
| Tokenizer | `transformers==5.13.1` / `tokenizers==0.22.2` | `@huggingface/transformers ^4.2.0` |

The ONNX wire format is stable across these minor versions. The risk is the **tokenizer.json format**: it can differ between `transformers` major versions. The tokenizer files shipped in `crossroad-model/` are generated by the Python version pinned above and must be consumable by `@huggingface/transformers` v4.x. If you upgrade `transformers` on either side, re-export the tokenizer and re-run the vitest eval to verify:

```powershell
$env:RUN_CROSSROAD_EVAL=1; npx vitest run src/tests/loop/crossroad-encoder-eval.test.ts
```

## config.json (output of train.py)

```json
{
  "version": 1,
  "baseModel": "distilbert-base-multilingual-cased",
  "maxSequenceLength": 512,
  "threshold": 0.7,
  "checkInterval": 50,
  "trainedAt": "2026-07-14T10:00:00Z",
  "trainingSamples": 500
}
```

| Field | Description |
|-------|-------------|
| `version` | Config format version |
| `baseModel` | HuggingFace base model name |
| `maxSequenceLength` | Max tokenizer sequence length |
| `threshold` | P(turn) threshold for triggering crossroad |
| `checkInterval` | How often (in chars) to run the detector during streaming |
| `trainedAt` | ISO 8601 timestamp of training |
| `trainingSamples` | Number of training samples used |

## Data Format

Each line in JSONL files:

**Positive** (has a turn):
```json
{"text": "I think we should use approach A. However, let me reconsider.", "turnIndex": 35, "label": 1}
```

**Negative** (no turn):
```json
{"text": "The approach is clean but requires more work.", "label": 0}
```

## Turning Patterns Covered

### English
- **Tier 1 (strong)**: However, Having said that, That being said, On the other hand, Hold on, On second thought, Then again
- **Tier 2 (sentence-boundary)**: Nevertheless, Nonetheless, That said, Actually, But (at sentence start)
- **Tier 3 (special)**: Wait (as interjection)

### Chinese
- **Strong**: 话说回来, 等一下, 但是
- **Sentence-boundary**: 然而, 但, 不过, 其实 (after 。！？； or newline)
- **Special**: 等等 (as "wait!" interjection), 不对 (as "no/wait" interjection)

### Non-turns (negatives)
- "But" mid-sentence (balanced analysis)
- "Wait for/until/to" (instructional verb)
- Chinese 等等 as "etc." (list terminator)
- Chinese 但/不过/其实 mid-sentence (conjunction)
- Normal coding/tool/file analysis outputs without turns
