# .crossroad-gen requirements — pinned to exact versions verified working
# with the v3 DistilBERT crossroad model (trained 2026-07).
#
# Versions captured from the successful v3 training environment
# (see pip-venv-install.log). Pinning prevents float/ops/API drift
# that silently breaks ONNX export or tokenizer format.
#
# ── Version contract with the mycc TypeScript runtime ──────────────
# The exported model.onnx is consumed by src/loop/crossroad-encoder.ts,
# which uses these npm packages:
#   onnxruntime-node      ^1.22.0   (Python: onnxruntime==1.27.0)
#   @huggingface/transformers ^4.2.0 (Python: transformers==5.13.1)
# The ONNX runtime wire format is stable across these minor versions,
# but the tokenizer.json format can differ between transformers major
# versions. The tokenizer files shipped in crossroad-model/ are
# generated by the Python transformers version below and must be
# consumable by @huggingface/transformers v4.x. If you upgrade
# transformers on either side, re-export the tokenizer and re-run
# the vitest eval (RUN_CROSSROAD_EVAL=1) to verify compatibility.
# ───────────────────────────────────────────────────────────────────
